# 超过宽度自提示组件

使用react hook 和antd 进行二次封装 可以根据是否超出元素宽度来进行提示的组件

const TipText = (props: { text: string }) => {
    const { text } = props;
    const ref = useRef < HTMLDivElement | null > ();
    const [open, setOpen] = useState(false);
    const setVisable = (open: boolean) => {
        if (open) {
            if (ref.current && ref.current.offsetWidth < ref.current.scrollWidth) {
                return setOpen(true);
            }
        }
        setOpen(false);
    };
    return (
        <Tooltip open={open} title={text}>
            <div
                ref={c => {
                    ref.current = c;
                }}
                className={styles.tiptext}
                onMouseOver={() => setVisable(true)}
                onMouseLeave={() => setVisable(false)}
              >
                {text}
            </div>
        </Tooltip>
    );
};

# react自定义hook

# 使用的场景

  1. 状态管理：自定义 Hooks 可以用于封装状态管理逻辑，==使多个组件能够共享和管理相同的状态==。例如，可以创建一个自定义 Hooks 用于处理全局应用状态、用户身份验证状态或者表单字段的状态管理。

  2. 副作用处理：自定义 Hooks 可以封装处理副作用操作的逻辑，如数据订阅、网络请求、本地存储等.==例如，可以创建一个自定义 Hooks 用于处理数据获取、定时器操作或者订阅事件的逻辑==。

  3. 数据获取和处理：自定义 Hooks 可以用于封装数据获取和处理的逻辑。例如，可以创建一个自定义 Hooks 用于从 API 中获取数据、对数据进行转换或者缓存数据。

  4. 表单处理：自定义 Hooks ==可以用于处理表单的逻辑，包括表单校验、表单提交、表单重置等==。例如，可以创建一个自定义 Hooks 用于处理表单校验和提交逻辑。

  5. 定时器和动画效果==：自定义 Hooks 可以用于处理定时器和动画效果的逻辑==。通过自定义 Hooks，可以集中处理定时器相关的逻辑，实现定时器的启动、暂停、停止等操作。同样，可以封装常见的动画逻辑，使其在多个组件中可复用。

  6. 访问浏览器 API：自定义 Hooks 可以用于封装访问浏览器 API 的逻辑，==如获取地理位置信息、访问本地存储、处理浏览器历史记录等==。

  7. 复杂逻辑的封装：自定义 Hooks 还可以用于封装处理复杂逻辑的代码块，==使其在多个组件中可复用。例如，可以创建一个自定义 Hooks 来处理分页逻辑、排序逻辑、权限控制逻辑等==，从而避免在多个组件中重复编写这些逻辑。

		// 返回验证错误信息

# 长按hook封装

interface ILongPressEventsProps {
  onLongPressCallback: (
    event: React.TouchEvent<HTMLDivElement | HTMLTextAreaElement | HTMLInputElement>['nativeEvent']['target']
  ) => void;
  onEndCallback?: (
    event: React.TouchEvent<HTMLDivElement | HTMLTextAreaElement | HTMLInputElement>['nativeEvent']['target']
  ) => void;
  delay?: number;
}

type ILongPressStartMethod = (event: React.TouchEvent) => void;
type ILongPressEndMethod = (event: React.TouchEvent) => void;
const useLongPress = ({ onLongPressCallback, onEndCallback, delay = 2000 }: ILongPressEventsProps) => {
  let timer: any = null;
  let target: EventTarget | null;
  let isStart = false;
  const start: ILongPressStartMethod = event => {
    if (event?.nativeEvent instanceof TouchEvent) target = event.nativeEvent.target;
    timer = setTimeout(() => {
      onLongPressCallback(target);
      isStart = true;
    }, delay);
    if (isStart) {
      event.stopPropagation();
      event.preventDefault();
    }
  };
  const stop: ILongPressEndMethod = event => {
    timer && clearTimeout(timer);
    if (event.nativeEvent instanceof TouchEvent) target = event.nativeEvent.target;
    if (isStart) {
      onEndCallback?.(target);
      event.stopPropagation();
      event.preventDefault();
      isStart = false;
    }
  };

  return {
    onTouchStart: start,
    onTouchMove: stop,
    onTouchEnd: stop,
  };
};

# 关于setState

关于setState的一些知识

1.setState是异步还是同步

  • 在合成事件和生命周期钩子内是异步的；
  • state和props不能保持一致性，会在开发中产生很多的问题；

由执行机制看，setState本身并不是异步的，而是如果在调用setState时，如果react正处于更新过程，当前更新会被暂存，等上一次更新执行后在执行，这个过程给人一种异步的假象

2.setState什么时候进行合并

3.setState需要使用不可变值

不可变值指的是已初始化的变量不允许修改，想要修改每次都要创建一个新的变量。

shouldMountUpdate生命周期中会将要改变的值与之前的值做个比较来确定是否改变视图，以这种方式来优化性能

shouldMountUpdate是浅比较，如果是修改原来的值，对象数据的地址是不会发生改变，视图就不会更新
